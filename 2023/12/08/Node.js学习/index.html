nodejs原型链污染

在JavaScript中，假设我们要定义一个类时，要以定义”构造函数”的方式来定义：

function Foo() {
	this.bar=1
}
new Foo()

其中Foo函数的内容，就是Foo类的构造函数，而this.bar就是Foo类的一个属性。在类当中也可以有一些方法，如下：

function Foo() {
    this.bar = 1
    this.show = function() {
        console.log(this.bar)
    }
}

(new Foo()).show()

每当我们新建一个类的时候，都会调用一次this.show这个方法，这个方法实际上是绑定在对象上的，就是new的这个对象上，而不是在类当中，此时，假设我们想要在创建类时只创建一次show方法，就得需要使用原型(prototype):

function Foo() {
    this.bar = 1
}

Foo.prototype.show = function show() {
    console.log(this.bar)
}

let foo = new Foo()
foo.show()

我们可以认为原型prototype是类Foo的一个属性，而所有用Foo类实例化的对象，都将拥有这个属性中的所有内容，包括变量和方法。比如上图中的foo对象，其天生就具有foo.show()方法。

我们可以通过Foo.prototype来访问Foo类的原型，但Foo实例化出来的对象，是不能通过prototype访问原型的。这时候，就该__proto__登场了。

一个Foo类实例化出来的foo对象，可以通过foo.__proto__属性来访问Foo类的原型.

属性遮蔽

如果你在一个对象中定义了一个属性，而在该对象的原型中定义了一个同名的属性,如下：

const myDate = new Date(1995, 11, 17);

console.log(myDate.getYear()); // 95

myDate.getYear = function () {
  console.log("别的东西！");
};

myDate.getYear(); // '别的东西！'

鉴于对原型链的描述，这应该是可以预测的。当我们调用 getYear() 时，浏览器首先在 myDate 中寻找具有该名称的属性，如果 myDate 没有定义该属性，才检查原型。因此，当我们给 myDate 添加 getYear() 时，就会调用 myDate 中的版本。

这叫做属性的“遮蔽”。

JavaScript 对象有一个指向一个原型对象的链。当试图访问一个对象的属性时，它不仅仅在该对象上搜寻，还会搜寻该对象的原型，以及原型的原型，依次层层向上搜索，直到找到一个名字匹配的属性或到达原型链的末尾。

const myDate = new Date();
let object = myDate;

do {
  object = Object.getPrototypeOf(object);
  console.log(object);
} while (object);

// Date.prototype
// Object { }
// null

这段代码创建了 Date 对象，然后遍历了它的原型链，记录并输出了原型。从中我们知道 myDate 的原型是 Date.prototype 对象，它（Date.prototype）的原型是 Object.prototype。

了解了什么是原型和原型链的逻辑之后，让我们来看什么是原型链污染。让我们举一个简单的例子：

var a = {number : 520}
var b = {number : 1314}
b.__proto__.number=520 
var c= {}
c.number

可以发现，c没有定义number属性，但我们依旧可以访问到520。就像我们上面说的，c没有找到number属性，就会向c.__proto__中寻找，假设载找不到，就会继续向上寻找，直到找到为止，又因为b.__proto__指向Object，而Object又是所有方法都存在的，故直接修改Object中的属性就可以达到原型链污染。

原型链被污染的情况

在实际中能够进行这种参数的函数一般有：

  • 对象合并（merge）；
  • 对象克隆（clone）（本质还是将一个对象合并到空对象中）

我们以合并为例，先搞一个merge函数：

function merge(target, source) {
   for (let key in source) {
       if (key in source && key in target) {
           merge(target[key], source[key])
      } else {
           target[key] = source[key]
      }
  }
}

合并的过程中存在赋值的操作：target[key] = source[key]那么我们将key改为__proto__是不是就可以原型链污染了呢？我们用代码来实验一下：

let o1 = {}
let o2 = {a: 1, "__proto__": {b: 2}}
merge(o1, o2)
console.log(o1.a, o1.b)
o3 = {}
console.log(o3.b)
console.log(o2)

var express = require('express');
var router = express.Router();
const isObject = obj => obj && obj.constructor && obj.constructor === Object;
const merge = (a, b) => {
  for (var attr in b) {
    if (isObject(a[attr]) && isObject(b[attr])) {
      merge(a[attr], b[attr]);
    } else {
      a[attr] = b[attr];
    }
  }
  return a
}
const clone = (a) => {
  return merge({}, a);
}

  if(keyword.match(/(admin)/is)) {
